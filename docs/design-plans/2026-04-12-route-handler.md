# Design Plan: `createRouteHandler` Abstraction

Every API route today repeats the same 4-step scaffold:

```ts
// src/app/api/contact/route.ts
export async function POST(request: NextRequest) {
  try {
    const { env } = getCloudflareContext();
    const db = getDb(env.DB); // ← duplicated
    const body = await request.json(); // ← duplicated
    const validatedData = schema.parse(body); // ← duplicated
    // ... feature logic ...
    return successResponse(result, "...", 201);
  } catch (error) { // ← duplicated
    if (error instanceof z.ZodError) {
      return validationErrorResponse(error, "...");
    }
    return errorResponse("...");
  }
}
```

This exact boilerplate exists in both `/api/contact/route.ts` and
`/api/pricing-estimate/route.ts`. As the project grows (admin, leads, tools,
billing) this will be copy-pasted across dozens of routes.

`createRouteHandler` eliminates this by extracting the scaffold into a typed,
composable factory while keeping business logic in feature modules where it
belongs.

---

## 2. Constraints & Environment

| Constraint       | Detail                                                                                             |
| ---------------- | -------------------------------------------------------------------------------------------------- |
| Runtime          | Cloudflare Workers via `@opennextjs/cloudflare`                                                    |
| DB               | Cloudflare D1 — only available from `getCloudflareContext()` at request time, never at module init |
| Validation       | Zod v4 (`zod@^4.3.6`)                                                                              |
| Response helpers | `~/lib/server/api-response.ts` — already exists, must reuse                                        |
| DB client        | `getDb(env.DB)` in `~/db/client.ts` — factory function, per-request                                |
| No Node APIs     | `fs`, `path`, `process.env` direct access not allowed in edge handlers                             |
| TypeScript       | Strict mode, `~/*` and `@/*` both alias `src/`                                                     |

---

## 3. API Design

### 3.1 Public surface

```ts
export const POST = createRouteHandler({
  schema: contactSchema, // optional Zod schema — infers `data` type
  successStatus: 201, // optional HTTP status for success (default: 200)
  use: [withLogging(), withAuth()], // optional middleware chain
  before: async (ctx) => {}, // optional lifecycle hook
  handler: async ({ data, db }) => {
    return await insertContactMessage(db, data);
  },
  after: async (ctx, result) => {}, // optional lifecycle hook
});
```

Returns a `(request: NextRequest) => Promise<NextResponse>` — a standard Next.js
App Router handler.

---

### 3.2 Context type

```ts
interface RouteContext<TData = unknown> {
  request: NextRequest;
  data: TData; // typed from schema via z.infer<TSchema>
  db: DbInstance; // ReturnType<typeof getDb>
  env: CloudflareEnv; // generated by `wrangler types`, globally ambient
}
```

`CloudflareEnv` is globally available — generated by `pnpm run cf-typegen` →
`cloudflare-env.d.ts`, included in `tsconfig.json`. No import needed.

---

### 3.3 Middleware type (Koa-style)

```ts
type Middleware<TData = unknown> = (
  ctx: RouteContext<TData>,
  next: () => Promise<NextResponse>,
) => Promise<NextResponse>;
```

Middleware either calls `next()` to continue the chain or returns a
`NextResponse` early (e.g. auth failure — 401 without calling handler).

---

### 3.4 Config type

```ts
interface RouteHandlerConfig<TSchema extends z.ZodTypeAny = z.ZodTypeAny> {
  schema?: TSchema;
  successStatus?: number;
  use?: Middleware<z.infer<TSchema>>[];
  before?: (ctx: RouteContext<z.infer<TSchema>>) => Promise<void>;
  handler: (ctx: RouteContext<z.infer<TSchema>>) => Promise<unknown>;
  after?: (
    ctx: RouteContext<z.infer<TSchema>>,
    result: unknown,
  ) => Promise<void>;
}
```

## 4. Execution Order

```
Request arrives
    │
    ▼
1. getCloudflareContext() → env, db
2. request.json() + schema.parse()  ← skipped if no schema
    │
    ▼  ZodError → validationErrorResponse(400)
3. Build RouteContext { request, data, db, env }
    │
    ▼
4. before(ctx)          ← optional, throws = error response
    │
    ▼
5. Middleware chain (use[0] → use[1] → ... → handler)
    │
    ├── Middleware calls next() → continues chain
    └── Middleware returns early → chain stops here
    │
    ▼  (inside core, at end of chain)
6. handler(ctx)         ← user business logic
    │
    ▼
7. after(ctx, result)   ← optional
    │
    ▼
8. successResponse(result, undefined, successStatus)
    │
    ▼
catch (error):
  ZodError       → validationErrorResponse(error)
  Error instance → errorResponse(error.message, 400)
  unknown        → errorResponse("An unexpected error occurred")
```

## 5. Implementation

### 5.1 Core file: `src/shared/lib/api/createRouteHandler.ts`

```ts
import type { NextRequest } from "next/server";
import { NextResponse } from "next/server";
import { z } from "zod";
import { getCloudflareContext } from "@opennextjs/cloudflare";
import { type DbInstance, getDb } from "~/db/client";
import {
  errorResponse,
  successResponse,
  validationErrorResponse,
} from "~/lib/server/api-response";

interface RouteContext<TData = unknown> {
  request: NextRequest;
  data: TData;
  db: DbInstance;
  env: CloudflareEnv;
}

type Middleware<TData = unknown> = (
  ctx: RouteContext<TData>,
  next: () => Promise<NextResponse>,
) => Promise<NextResponse>;

interface RouteHandlerConfig<TSchema extends z.ZodTypeAny = z.ZodTypeAny> {
  schema?: TSchema;
  successStatus?: number;
  use?: Middleware<z.infer<TSchema>>[];
  before?: (ctx: RouteContext<z.infer<TSchema>>) => Promise<void>;
  handler: (ctx: RouteContext<z.infer<TSchema>>) => Promise<unknown>;
  after?: (
    ctx: RouteContext<z.infer<TSchema>>,
    result: unknown,
  ) => Promise<void>;
}

function compose<TData>(
  middlewares: Middleware<TData>[],
  core: (ctx: RouteContext<TData>) => Promise<NextResponse>,
): (ctx: RouteContext<TData>) => Promise<NextResponse> {
  return (ctx) => {
    let index = 0;
    const next = (): Promise<NextResponse> => {
      if (index >= middlewares.length) return core(ctx);
      return middlewares[index++](ctx, next);
    };
    return next();
  };
}

export function createRouteHandler<TSchema extends z.ZodTypeAny>(
  config: RouteHandlerConfig<TSchema>,
): (request: NextRequest) => Promise<NextResponse> {
  return async (request: NextRequest): Promise<NextResponse> => {
    try {
      const { env } = await getCloudflareContext();
      const db = getDb(env.DB);

      let data = undefined as z.infer<TSchema>;
      if (config.schema) {
        const body = await request.json();
        data = config.schema.parse(body);
      }

      const ctx: RouteContext<z.infer<TSchema>> = { request, data, db, env };

      if (config.before) {
        await config.before(ctx);
      }

      const core = async (
        ctx: RouteContext<z.infer<TSchema>>,
      ): Promise<NextResponse> => {
        const result = await config.handler(ctx);
        if (config.after) {
          await config.after(ctx, result);
        }
        return successResponse(
          result,
          undefined,
          config.successStatus ?? 200,
        ) as NextResponse;
      };

      const chain = compose(config.use ?? [], core);
      return await chain(ctx);
    } catch (error) {
      console.error("[createRouteHandler]", error);

      if (error instanceof z.ZodError) {
        return validationErrorResponse(error) as NextResponse;
      }
      if (error instanceof Error) {
        return errorResponse(error.message, 400) as NextResponse;
      }
      return errorResponse("An unexpected error occurred") as NextResponse;
    }
  };
}

export type { Middleware, RouteContext, RouteHandlerConfig };
```

### 5.2 Middleware factories: `src/shared/lib/api/middleware/`

**`withLogging.ts`**

```ts
import type { Middleware } from "../createRouteHandler";

export function withLogging(): Middleware {
  return async (ctx, next) => {
    const start = Date.now();
    const { method, url } = ctx.request;
    const response = await next();
    console.log(
      `[${method}] ${new URL(url).pathname} — ${
        Date.now() - start
      }ms ${response.status}`,
    );
    return response;
  };
}
```

**`withAuth.ts`** (session-based, using Better Auth's `getSessionCookie`
pattern)

```ts
import { getSessionCookie } from "better-auth/cookies";
import { errorResponse } from "~/lib/server/api-response";
import type { Middleware } from "../createRouteHandler";

export function withAuth(): Middleware {
  return async (ctx, next) => {
    const cookie = getSessionCookie(ctx.request);
    if (!cookie) {
      return errorResponse("Unauthorized", 401) as any;
    }
    return next();
  };
}
```

## 6. Migration: `contact/route.ts` (before → after)

**Before (49 lines):**

```ts
export async function POST(request: NextRequest) {
  try {
    const { env } = getCloudflareContext();
    const db = getDb(env.DB);
    const body = await request.json();
    const validatedData = contactSchema.parse(body);
    const insertedMessage = await insertContactMessage(db, validatedData);
    logContactSubmission(insertedMessage);
    try {
      await sendEmailNotification(insertedMessage);
    } catch (emailError) {
      console.error("Email notification failed:", emailError);
    }
    return successResponse({ id: insertedMessage.id }, "Thank you...", 201);
  } catch (error) {
    if (error instanceof z.ZodError) {
      return validationErrorResponse(error, "Invalid form data");
    }
    return errorResponse(
      "Failed to submit contact form. Please try again later.",
    );
  }
}
```

## 7. GET Route Support

When no `schema` is provided, body parsing is skipped. `data` will be
`undefined`. This supports stateless GET handlers:

```ts
export const GET = createRouteHandler({
  handler: async ({ request, db }) => {
    const url = new URL(request.url);
    const id = url.searchParams.get("id");
    return await getRecord(db, id);
  },
});
```

## 10. Key Design Decisions & Trade-offs

| Decision                                              | Rationale                                                                   |
| ----------------------------------------------------- | --------------------------------------------------------------------------- |
| Koa-style onion middleware (not Express)              | Works without Node — pure async functions, no `res.send()`                  |
| `await getCloudflareContext()` (not sync)             | `pricing-estimate` already uses `await`; safe for both sync/async return    |
| `successStatus` config field                          | Routes need 201 for POST creation — handler shouldn't know about HTTP       |
| `after` runs inside `core`, inside the chain          | Middleware wrapping the `core` can observe `after` side effects if needed   |
| Error from `before`/`after` caught at top level       | Consistent error response regardless of which lifecycle step throws         |
| No auto-wrapping of `after` errors                    | `after` is for fire-and-forget side effects; errors bubble to main catch    |
| `CloudflareEnv` globally ambient                      | Generated by `cf-typegen`, referenced in `tsconfig.json` — no import needed |
| `data = undefined as z.infer<TSchema>` when no schema | Avoids conditional type branching; GET routes can ignore `data`             |
